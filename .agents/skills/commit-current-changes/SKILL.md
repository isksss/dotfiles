# commit-current-changes

このスキルは、現在の Git 変更を確認し、簡潔な prefix 付き日本語コミットメッセージで意図の明確なコミットにするために使う。

## 手順

1. ステージ前にリポジトリ状態を確認する。
   - `git status --short --branch` を実行する。
   - `git diff --stat` と `git diff --cached --stat` を実行する。
   - 必要に応じ `git diff --name-only` と `git diff --cached --name-only` を確認する。
   - 必要な差分を `git diff` と `git diff --cached` で読む。
   - 未追跡ファイルは、ステージ前にファイル名と内容を確認する。
   - rename、binary、permission change を見落とさない。

2. コミット範囲を決める。
   - 既定では現在の全変更を確認する。
   - 生成物、キャッシュ、依存関係、ログ、ローカル環境ファイル、秘密情報は除外する。
   - `.env`、`.env.*`、`node_modules`、`.nuxt`、`.output`、`.nitro`、`.cache`、`dist` などのローカル生成物は含めない。
   - `.gitignore` に含まれるべき生成物が差分に存在する場合、コミット前に確認する。
   - 機能追加・リファクタ・フォーマット変更が混在する場合、可能なら分割コミットする。
   - 無関係な変更が混ざっている場合、実用的なら分割コミットを優先する。分割しない場合は、まとめる範囲を説明する。
   - ユーザーが明示的に依頼しない限り、ユーザー変更を revert・破棄しない。

3. prefix 付き日本語コミットメッセージを作成する。
   - 1行目は `[prefix] 日本語の要約` 形式にする。
   - prefix は変更内容に応じて選択する。
   - `[add]`: 新機能、新ファイル、新設定追加。
   - `[fix]`: 不具合修正。
   - `[update]`: 既存機能、設定、文言更新。
   - `[remove]`: 不要コードやファイル削除。
   - `[refactor]`: 挙動変更なし整理。
   - `[docs]`: ドキュメントのみ変更。
   - `[test]`: テスト追加・修正。
   - `[chore]`: ビルド、依存関係、開発補助など保守作業。
   - `[perf]`: パフォーマンス改善。
   - `[ci]`: CI/CD 設定変更。
   - 1行目はユーザーに見える成果または保守上成果を簡潔に要約する。
   - 「更新」「修正」だけの曖昧な要約にしない。
   - 「何を」「どうした」を含める。
   - 主対象と操作を含める。
   - 40〜60文字程度推奨。
   - 例:
     - `[update] Nuxt MCP の設定 URL を更新`
     - `[add] コミット作成用 Codex skill を追加`
   - 差分に補足説明が必要な場合だけ本文追加する。
   - 破壊的変更、移行必要、設定変更時のみ本文追加推奨。

4. ステージしてコミットする。
   - 選択したファイルだけステージする。
   - `git add .` の無条件使用を避ける。
   - パス指定で必要ファイルのみステージする。
   - package.json に `lint` や `typecheck` が存在し、数分以内で完了見込みなら実行する。
   - lint や typecheck を実行しない場合、その理由を説明する。
   - pre-commit hook が失敗した場合、内容確認し、自動修正可能なら修正後再実行する。
   - サンドボックス制約で Git メタデータが読み取り専用となり `git add` や `git commit` が失敗した場合、必要承認付きで同じコマンドを再実行する。
   - `git` コマンドが読み取り専用メタデータ以外理由で失敗した場合、エラー内容を報告し、可能な解決策を提示する。
   - 選択した prefix 付き日本語メッセージでコミットする。
   - コミットに `Co-authored-by: Copilot` を付与しない。

5. 確認して報告する。
   - `git status --short` を実行する。
   - `git log -1 --oneline` を実行する。
   - コミットハッシュとコミットメッセージを日本語で報告する。
   - 必要ならコミット対象概要を 1〜3 項目で要約する。
   - 未コミット変更が残る場合、何がなぜ残っているか明確に説明する。

## 安全確認

- コミット前に、認証情報、トークン、秘密鍵、個人情報、マシン固有パスが差分に含まれていないか確認する。
- 秘密情報や private ファイル可能性があるものを見つけた場合、作業停止し、進め方を確認する。
- 意図しない生成物や巨大ファイルが含まれていないか確認する。
- force push 前提変更や履歴改変必要な操作は、ユーザー明示依頼なし実施しない。

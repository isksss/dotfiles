# implementation

この skill は、調査結果または計画に基づいて実装を進めるために使う。

## 参照ルール

- `.agents/rules/implementation.md`
- `.agents/rules/development.md`
- `.agents/rules/agent-separation.md`
- `.agents/rules/safety.md`

## 手順

1. 前提を確認する。
   - ユーザー依頼、計画、調査メモ、既存差分を読む。
   - 作業ツリーにユーザー変更がある場合、壊さず共存する。

2. 実装範囲を決める。
   - 要求範囲に必要なファイルだけ変更する。
   - 無関係な整形、リファクタ、依存追加を避ける。
   - 新しい抽象化は必要性が明確な場合だけ追加する。

3. 変更する。
   - 既存スタイル、命名、ファイル構成に合わせる。
   - エラー処理、境界条件、後方互換性を確認する。
   - コメントは必要最小限にする。

4. 検証へ引き継ぐ。
   - 変更内容、想定影響範囲、実行すべき検証を整理する。
   - 可能な検証は実行し、レビュー担当が確認しやすい状態にする。

5. 記録する。
   - 長時間作業、引き継ぎ、重要判断がある場合は
     `{repo_root}/note.local.dir/agent/{yyyymmdd-hhmm}_implementation.md` に記録する。
   - 軽微な実装では最終回答で代替してよい。
   - 目的、変更内容、検証結果、残作業を書く。
   - 秘密情報、認証情報、個人情報は記録しない。

## 成果物

- 要求範囲に限定された実装差分。
- レビュー担当が確認しやすい変更概要と検証メモ。

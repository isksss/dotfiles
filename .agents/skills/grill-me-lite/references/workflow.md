# grill-me-lite workflow

## 事前調査

質問前に関連する README、設定、既存実装、テストを確認する。
判明した事実は質問せず、短く報告する。

## ヒアリング方針

- 一度に質問する内容は1件のみ。
- 最大5〜7問を目安にする。
- 固定順で消化せず、実装判断に最も効く未決定事項から聞く。
- 甘い回答や曖昧な回答には追加で1段だけ突っ込む。
- 実装、コード生成、Issue生成へ進まない。

## 確認観点

- 目的: 何を達成したいか。
- 成功条件: どうなれば完了か。
- 利用者: 誰が使うか。
- 業務フロー: 開始条件、主要操作、完了条件。
- 入出力: 入力、出力、エラー。
- 受入条件: 確認可能な条件。

## 質問フォーマット

```markdown
### Q<番号>: <質問文>

<なぜこの質問が重要か>

- A: <選択肢>
- B: <選択肢>

**推奨: <A/B>** — <理由>
```

## 終了判断

- 目的、成功条件、主要フロー、受入条件が実装可能な粒度になった。
- 残った未確認事項が小さく、実装方針を左右しない。
- 5〜7問に達し、追加深掘りの費用対効果が低い。

## 完了時出力

```markdown
## まとめ

### 決まったこと

- ...

### 決定メモ

- ...

### 受入条件

- ...

### 未解決事項

なし

仕様確定。実装へ進む？
```

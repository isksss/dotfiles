# Long Running Task Rules

## when to use

- 調査、実装、検証が長時間化する見込みのとき。
- 複数ステップ、複数ファイル、外部ツール実行を伴うとき。

## 基本方針

- 目的、現在地、残作業、既知のリスクを短く維持する。
- 30秒以上進展共有がない場合、進捗または待機理由を伝える。
- 長時間コマンドは完了まで放置せず、必要に応じて途中出力を確認する。
- 中断や再開に備え、判断理由と検証結果を記録する。

## 記録

- 長時間作業、引き継ぎ、重要判断がある場合は、調査内容、目的、
  現在地、残作業、検証結果を `{repo_root}/note.local.dir/agent/` に記録する。
- 軽微な作業では最終回答で代替してよい。
- ファイル名は `{yyyymmdd-hhmm}_{summary}.md` とする。
- 秘密情報、認証情報、個人情報は記録しない。

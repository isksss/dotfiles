# Research Rules

## when to use

- 実装前の調査、原因分析、仕様確認、影響範囲確認を行うとき。
- コード、ドキュメント、設定、外部仕様を根拠として判断するとき。

## 調査方針

- 推測より、コード、設定、ドキュメント、実行結果を優先する。
- まず `rg`、`git diff`、関連 entrypoint、設定ファイルを確認する。
- 見つかった事実、未確認事項、実装への影響を分けて整理する。
- 外部情報が必要な場合は一次情報を優先し、参照日と URL を残す。
- 調査段階では原則として repo-tracked file を変更しない。

## 成果物

- 目的、確認したファイル、判明事項、未解決事項、次の実装方針を記録する。
- 実装担当が判断に迷わない粒度で、必要最小限にまとめる。

# Communication Rules

## when to use

- ユーザーへの回答、途中経過共有、最終報告を書くとき。
- 複数案、未検証事項、実行不能事項を説明するとき。

## 基本方針

- 思考言語 任意
- 最終回答 日本語
- 回答 簡潔・明確・実行可能優先
- 推測より事実優先
- 不明点 明示
- 必要時 根拠・理由・影響範囲 併記
- 既存実装・既存規約 尊重
- 変更最小化優先

## 回答ルール

- 要求範囲のみ回答
- 不必要な前置き・長文説明 禁止
- コード変更時 下記優先:
  1. 原因特定
  2. 影響範囲確認
  3. 最小修正
  4. 検証
- 複数案ある場合:
  - 推奨案
  - メリット
  - デメリット
  を簡潔提示
- コマンド・パス・識別子 原文維持
- Markdown 使用可
- 不要な絵文字 禁止

## 出力形式

### コード変更時

`.agents/templates/code-change-report.md` を優先する。

### コマンド提示時

`bash`形式で提示。

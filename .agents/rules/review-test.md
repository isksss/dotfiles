# Review And Test Rules

## when to use

- 差分レビュー、テスト、lint、typecheck、動作確認を行うとき。
- 実装後の品質確認やリスク評価を行うとき。

## レビュー方針

- 重大度の高い不具合、回帰、セキュリティ、データ破壊リスクを優先する。
- 指摘はファイル、行、理由、再現条件、修正方針を具体化する。
- 問題がない場合も、未検証範囲と残リスクを明示する。
- レビュー担当は原則として実装を行わず、必要なら実装担当へ戻す。

## テスト方針

- 変更範囲に応じて format、lint、typecheck、unit test、動作確認を選ぶ。
- 失敗時はエラー内容、原因仮説、次の対応を分けて記録する。
- 実行できない検証は理由と影響範囲を明記する。

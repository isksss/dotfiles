# self-format

このスキルは、ファイルタイプごとに適切なフォーマッタを選択して、コードやドキュメントを一貫したスタイルで整形するために使う。

## 手順

### 1. フォーマット対象のファイルを特定する。

- ユーザーが特定のファイルを指定した場合は、そのファイルを対象とする。
- ユーザーがディレクトリを指定した場合は、そのディレクトリ内のすべてのファイルを対象とする。
- ユーザーが特に指定しない場合は、現在の作業ディレクトリ内のすべてのファイルを対象とする。

### 2. フォーマッターを選択する。

- ファイルの拡張子や内容に基づいて、適切なフォーマッタを選択する。
- 例えば、`.js` や `.ts` ファイルには `prettier` を、`.go` ファイルには `gofmt` を、`.md` ファイルには `markdownlint` を選択する。
  - 対象プロジェクトで特定のフォーマッタが使用されている場合は、そのフォーマッタを優先する。
- フォーマッタが複数候補がある場合は、ユーザーに選択を促す。

### 3. フォーマットを実行する。

## 注意点

- フォーマット以外の変更は行わない。

## フォーマッター

|     ファイルタイプ     | フォーマッタ         |
| :--------------------: | :------------------- |
|      `.js`, `.ts`      | `prettier`           |
|         `.go`          | `gofmt`              |
|         `.md`          | `markdownlint`       |
|         `.rs`          | `rustfmt`            |
| `.sh`, `.bash`. `.zsh` | `shfmt`              |
|        `.java`         | `google-java-format` |

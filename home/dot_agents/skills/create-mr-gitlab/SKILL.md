# create-mr-gitlab

## 概要

`glab`コマンドを使用して、GitLab でマージリクエストを作成するためのスキルです。ユーザーが GitLab でのマージリクエスト作成を依頼したときに使用します。

このスキルは、現在の作業ブランチから「現在ブランチの派生元ブランチ」への MR 作成を既定とし、必要に応じてマージ先を変更して実行します。

## 手順

### 1. 現在のブランチを確認する

- `git branch --show-current` を実行して、現在のブランチ名を取得します。
- ブランチ名をユーザーに報告します。
- ブランチ名が取得できない場合は、エラーをユーザーに報告し、マージリクエストの作成を中止します。
- ブランチ名が `main`、`master`、`develop` などの主要なブランチである場合は、マージリクエストの作成を中止し、ユーザーに警告します。

実行例:

```sh
git branch --show-current
```

### 2. 未コミットの内容がないか確認する

- `git status --short` を実行して、未コミットの変更がないか確認します。
- 変更がある場合は、ユーザーに報告し、マージリクエストの作成を中止します。
- 変更がない場合は、次のステップに進みます。

実行例:

```sh
git status --short
```

加えて、MR 本文生成に使うコミット履歴を確認します。

```sh
BASE_BRANCH="$(git reflog show --format='%gs' "$(git branch --show-current)" | sed -n 's/^branch: Created from //p' | head -n1)"
git log --oneline "origin/${BASE_BRANCH}..HEAD"
```

- `BASE_BRANCH` が取得できない、または `HEAD` になる場合は、派生元を自動判定できないためユーザーに確認します。
- `origin/${BASE_BRANCH}..HEAD` が空の場合は、MR に含める変更がないため作成を中止します。

### 3. マージリクエストを作成する

- タイトルと説明は現在のブランチのコミット履歴から生成します。
  - タイトルにはprefixを付けることが望ましいです。例: `[feature] 新しい機能の追加`、`[fix] バグ修正`、`[update] 既存機能の更新`。
  - タイトルは `origin/${BASE_BRANCH}..HEAD` の全コミットメッセージを読み、変更全体を1行で要約して作成します。
  - 説明には、`origin/${BASE_BRANCH}..HEAD` のコミットメッセージ一覧を引用できます。

手順:

- `glab` が利用可能か確認します。

```sh
command -v glab >/dev/null 2>&1
```

- 未ログイン時は `glab auth status` で状態確認し、必要ならユーザーにログインを依頼します。

```sh
glab auth status
```

- タイトルは「全コミット履歴の共通テーマを要約した prefix + 要約」にします。
- 本文は次のテンプレートを基本に、`origin/${BASE_BRANCH}..HEAD` のコミット一覧を添えて作成します。

```md
## 概要

- （変更の要点を 1-3 行で記述）

## 変更内容

- （主要変更1）
- （主要変更2）

## コミット

- <hash> <message>
- <hash> <message>
```

- MR を作成します（既定のマージ先は `BASE_BRANCH`）。

```sh
glab mr create \
  --source-branch "$(git branch --show-current)" \
  --target-branch "${BASE_BRANCH}" \
  --title "[update] （要約）" \
  --description "（生成した本文）" \
  --fill=false
```

- ユーザーからマージ先ブランチ指定がある場合は `--target-branch` をその値に変更します。

### 4. マージリクエストのURLをユーザーに報告する

- `glab mr create` の出力に含まれる URL を抽出して報告します。
- 必要に応じて、作成された MR の確認として次を実行します。

```sh
glab mr view --web
```

## 注意事項

- マージリクエストの作成には、ユーザーが `glab` コマンドを使用して GitLab にアクセスできる環境が必要です。
- ブランチ名が主要なブランチである場合は、マージリクエストの作成を中止し、ユーザーに警告します。
- 未コミットの変更がある場合は、マージリクエストの作成を中止し、ユーザーに報告します。
- マージリクエストは、現在のブランチから派生元ブランチへのマージを前提とします。派生元を自動判定できない場合は、必ずユーザーにマージ先を確認してください。
- `glab` 未導入・未認証・権限不足で失敗した場合は、エラー内容をそのまま共有し、認証または権限設定後に再実行します。
